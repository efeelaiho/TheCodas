Data

# Data format

In order to gather data, a python script was created, taking advantage of several APIs and some Python libraries that made integration of the different APIs and formatting simpler.

The data was extracted from Spotify.com using their Web API1. Using a Python library called Spotipy2, the artist data and album data were fetched from the Spotify music catalog with a Python script isolating the information that was relevant to the models from the entirety of Spotify Web API such as artist and album names, artist and album URIs, album images, album release date, album artist name, and artist popularity.

A Pythonic wrapper named Wikipedia3 was also used in conjuction with a parser called BeautifulSoup4 to extract the summary section of the artists' Wikipedia pages to be used as bio sections for the artists.

A parser called mwparserfromhell5 was used to parse the infoboxes of the artist pages in order to find a person's birthplace or a group's origin.

For the Wikipedia information, regular expressions had to be used in order to create the correctly formatted search queries and to make the extracted information look cleaner.

For artist images, the images that are pulled by Spotify varied in dimension, so instead, Google Image Search API6 was used. Though deprecated, Google Image Search API was able to fetch the first several images of a Google image search query of Spotify <artist name>, from which the first image with a 640x640 size was extracted.

All the data was stored in a single json object as a list of dictionaries, with primary keys being generated as the data was scraped.

An artist dictionary contains dictionaries of their primary key, model, prview track URL (to be rendered as a preview audio player at the top of artist pages), and a nested dictionary of fields with origin, popularity, recommended album (the album of the the artist's top track), image URL (first 640x640 image returned from a Google image search query of Spotify <artist name>), Spotify artist URI, full name, two YouTube video URLs, genre, and biography (Summary from the artist's Wikipedia page).

Currently, the file contains the top 50 top selling artists, according to Wikipedia, plus 65 artists personally hand selected by the team. The albums are each listed artist's top track's album, plus three albums personally hand selected by the team.

# Accessing the data

# Internal use

In order to access the data internally, the cache file created was used to populate the MySQL database. To populate the SQL database, the python manage.py loaddata finalcache.json command may be run. The only difficulty is that Django expects the .json file to be in a particular serialized format which matches the models.py file, which can be seen in the above code snippets.

Note that all the elements from the inner "fields" dictionary are variable names in the Artist class from models.py. "model" tells Django which class to use (volumemax.Artist or volumemax.Album for this project), and "pk" means "primary key" and is used to reference this item in as a foreign key, for example. A foreign key (as in the Album.album_artist field) takes as an integer value the pk where the artist is found.
