# Data format

The data was extracted from Spotify.com using their Web API<sup>1</sup>. Using a Python library called Spotipy<sup>2</sup>, the artist data and album data were fetched from the Spotify music catalog with a Python script isolating the information that was relevant to the models (and some data that will hopefully be utilized in future phases for a richer user expererience). A Pythonic wrapper named Wikipedia<sup>3</sup> was also used to extract the summary section of the artists' and albums' Wikipedia pages. All the data was stored in a single json object as a dictionary indexed by their Spotify IDs.

The artist dictionary contains dictionaries of their name, top track names, dictionaries of album names and their album art, Wikipedia page summary, and the url of the artist image. For example:

{"19eLuQmk9aCobbVDHc6eek": 
    {"toptracks": 
        ["What A Wonderful World", "Dream A Little Dream Of Me", "La Vie En Rose - Single Version"],
     "albums": ["By Popular Demand", "In Person", ...], 
     "img": "https://i.scdn.co/image/7e61be280d4f457b89d563be6b2a11e8ef9e67f3"}, 
"3fMbdgg4jU18AjLCKBhRSm": 
    {"toptracks": 
        ["Billie Jean - Single Version", "Love Never Felt so Good", "Love Never Felt so Good"], 
     "albums": ["XSCAPE", "Michael", "Invincible", "BLOOD ON THE DANCE FLOOR/ HIStory In The Mix", ...],
     "img": "https://i.scdn.co/image/738650ce127e119788a7bca020fbd054b5aa57b5"}, ...}

This is stored in TheCodas/vmax_src/artistcache.json (The artist cache will eventually hold more information: birth and death and/or years active information, genres, birthplace/origin, members, website, the top tracks will be a dictionary of track names keyed to their Spotify track IDs, and possibly more)

The album dictionary contains dictionaries of their name, list of tracks and the url of the album art. For example:

{"6YlPSTWp7bq3Q01L9EcaGG": 
    {"name": "Heart Full Of Rhythm / Volume 2 (1936-38)", 
     "img": "https://i.scdn.co/image/6b843f9e2a1df891f5c270577c056621f50d5413", 
     "albumtracks": ["I Come From A Musical Family - Single Version", "Somebody Stole My Break - Single Version", "If We Never Meet Again - Single Version", ...]},
 "5xifZlByZcZecgkP7sYSoW": 
     {"name": "The Marshall Mathers LP", 
      "img": "https://i.scdn.co/image/2cdee8b0dd96ca108db92f3894366c574055f959", 
      "albumtracks": ["Public Service Announcement 2000", "Kill You", "Stan", "Paul - Skit (Explicit)", "Who Knew", "Steve Berman", ...]}, ...}

This is stored in TheCodas/vmax_src/albumcache.json (The album cache will eventually hold more information: release date, genres, the track list will be a dictionary of track names keyed to their Spotify track IDs, and possibly more.)

Currently, these files contain sample data of just six artists and their albums:

  • Louis Armstrong
  • Miles Davis
  • Bill Evans
  • Eminem
  • Kanye West
  • Michael Jackson

However, the plan is to expand this to the top 50 artists, found from scraping another (to be determined if not available in Spotify Web API) API and processing that data through the python script.

# Accessing the data

# Internal use

In order to access the data internally, the JSON dictionaries will be loaded as cached data. The data can then be called throughout the application readily by key and/or index.

# External use

The data is exposed using the apiary API volumemin<sup>4</sup>. It can be accessed at http://volumemax.me:8000/api . See the apiary documentation for more details. For example:

http://volumemax.me:8000/api/artists/1

{ "toptracks": ["What A Wonderful World", "Dream A Little Dream Of Me", "La Vie En Rose - Single Version"], "albums": ["By Popular Demand", "In Person"], "img": "https://i.scdn.co/image/7e61be280d4f457b89d563be6b2a11e8ef9e67f3" }
